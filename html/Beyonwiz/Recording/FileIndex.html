NAME

    use Beyonwiz::Recording::FileIndex;

SYNOPSIS

Provides access to the Beyonwiz recording file index via HTTP.

SUPERCLASS

Inherits from Beyonwiz::Recording::Index

METHODS

Beyonwiz::Recording::FileIndex->new($base [, $makeSortTitle])

Create a new Beyonwiz recording index object. $base is the base URL for the Beyonwiz device. $makeSortTitle takes a single string argument, and its return value is used to construct Beyonwiz::Recording::IndexEntry::sortTitle;. It should transform its input string to the form used for comparisons when sorting (for example in $i->entries([$val]);)

$i->path([$val]);

Returns (sets) the recording path.

$i->newEntry($name, $path, $makeSortTitle);

Create a new Beyonwiz::Recording::HTTPIndexEntry.

$i->load;

Load the index from the Beyonwiz.

PREREQUISITES

Uses packages: Beyonwiz::Recording::Header, Beyonwiz::Recording::Trunc, Beyonwiz::Recording::Index, Beyonwiz::Recording::FileIndexEntry, Beyonwiz::Utils, LWP::Simple, URI, URI::Escape, File::Find, File::Spec::Functions.

BUGS

Uses a fixed value for the path name of the index, rather than deriving it from locationURL in Beyonwiz::WizPnP.
